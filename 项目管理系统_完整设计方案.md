## 📋 一、系统概述

### 1.1 项目背景
为了更好地管理个人电脑上的多个开发项目，需要一个集中化的项目管理系统，能够：
- 📊 统一管理所有项目信息
- 📅 追踪每个项目的进度和时间线
- 📝 记录详细的开发日志（类似Git提交记录）
- 📈 可视化展示项目统计数据

### 1.2 核心价值
- ✅ 清晰管理多个项目进度
- ✅ 记录开发历程（类似个人开发日记）
- ✅ 可视化学习成长轨迹
- ✅ 提高时间管理能力
- ✅ 展示项目经历和技术积累

**设计理念：** 参考Git Commit Log，但更加直观和丰富

  ├─ 14:30 [功能] 完成用户登录模块
  │   └─ 详情：实现JWT认证、记住密码功能

  │

  │   └─ 详情：解决移动端导航栏响应式布局bug
  │   └─ 代码变更：+12行 -8行

  └─ 18:20 [文档] 更新API文档

📅 2024年11月29日 周五

- 📝 **[文档]** - 文档更新
- 🧪 **[测试]** - 测试相关
- 🚀 **[部署]** - 部署、发布
- 🔧 **[配置]** - 配置文件修改
- 💡 **[想法]** - 待实现的想法记录

## 🗄️ 五、数据库设计

### 5.1 核心表结构

#### 1. 项目表 (projects)
```sql
CREATE TABLE projects (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(200) NOT NULL COMMENT '项目名称',
  description TEXT COMMENT '项目描述',
  cover_image VARCHAR(500) COMMENT '封面图片URL',
  status VARCHAR(20) DEFAULT '进行中' COMMENT '项目状态：进行中/暂停/已完成/计划中',
  progress INT DEFAULT 0 COMMENT '项目进度 0-100',
  local_path VARCHAR(500) COMMENT '本地路径',
  repo_url VARCHAR(500) COMMENT '仓库URL',
  tech_stack JSON COMMENT '技术栈数组',
  members JSON COMMENT '项目成员',
  expected_end_date DATE COMMENT '预计完成时间',
  is_favorite BOOLEAN DEFAULT FALSE COMMENT '是否收藏',
  sort_order INT DEFAULT 0 COMMENT '排序',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_status (status),
  INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='项目表';
```

#### 2. 时间轴记录表 (timeline_logs)
```sql
CREATE TABLE timeline_logs (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  project_id BIGINT NOT NULL COMMENT '项目ID',
  log_type VARCHAR(20) NOT NULL COMMENT '记录类型：功能/修复/优化/文档/测试/部署/配置/想法/里程碑',
  title VARCHAR(200) NOT NULL COMMENT '标题',
  description TEXT COMMENT '详细描述',
  code_additions INT DEFAULT 0 COMMENT '代码新增行数',
  code_deletions INT DEFAULT 0 COMMENT '代码删除行数',
  images JSON COMMENT '相关图片数组',
  git_commit_hash VARCHAR(100) COMMENT 'Git提交哈希',
  is_milestone BOOLEAN DEFAULT FALSE COMMENT '是否里程碑',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
  INDEX idx_project_id (project_id),
  INDEX idx_created_at (created_at),
  INDEX idx_log_type (log_type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='时间轴记录表';
```

#### 3. 任务表 (tasks)
```sql
CREATE TABLE tasks (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  project_id BIGINT NOT NULL COMMENT '项目ID',
  title VARCHAR(200) NOT NULL COMMENT '任务标题',
  description TEXT COMMENT '任务描述',
  status VARCHAR(20) DEFAULT '待办' COMMENT '状态：待办/进行中/已完成',
  priority VARCHAR(20) DEFAULT '中' COMMENT '优先级：高/中/低',
  estimated_hours DECIMAL(10,2) DEFAULT 0 COMMENT '预计工时',
  actual_hours DECIMAL(10,2) DEFAULT 0 COMMENT '实际工时',
  deadline DATE COMMENT '截止日期',
  tags JSON COMMENT '标签数组',
  sort_order INT DEFAULT 0 COMMENT '排序',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
  INDEX idx_project_id (project_id),
  INDEX idx_status (status),
  INDEX idx_deadline (deadline)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='任务表';
```

#### 4. 子任务表 (sub_tasks)
```sql
CREATE TABLE sub_tasks (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  task_id BIGINT NOT NULL COMMENT '父任务ID',
  title VARCHAR(200) NOT NULL COMMENT '子任务标题',
  completed BOOLEAN DEFAULT FALSE COMMENT '是否完成',
  sort_order INT DEFAULT 0 COMMENT '排序',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (task_id) REFERENCES tasks(id) ON DELETE CASCADE,
  INDEX idx_task_id (task_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='子任务表';
```

#### 5. 每日日志表 (daily_logs)
```sql
CREATE TABLE daily_logs (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  log_date DATE NOT NULL UNIQUE COMMENT '日志日期',
  content TEXT COMMENT '日志内容',
  mood VARCHAR(20) COMMENT '心情：开心/一般/沮丧',
  work_state VARCHAR(20) COMMENT '工作状态：高效/正常/低效',
  work_hours DECIMAL(10,2) DEFAULT 0 COMMENT '工作时长',
  achievements JSON COMMENT '当日成就数组',
  tomorrow_plan TEXT COMMENT '明日计划',
  related_projects JSON COMMENT '关联项目ID数组',
  related_tasks JSON COMMENT '关联任务ID数组',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_log_date (log_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='每日日志表';
```

#### 6. 标签表 (tags)
```sql
CREATE TABLE tags (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(50) NOT NULL UNIQUE COMMENT '标签名称',
  color VARCHAR(20) DEFAULT '#409EFF' COMMENT '标签颜色',
  category VARCHAR(50) COMMENT '标签分类：技术栈/项目类型/其他',
  usage_count INT DEFAULT 0 COMMENT '使用次数',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_category (category)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='标签表';
```

#### 7. 项目-标签关联表 (project_tags)
```sql
CREATE TABLE project_tags (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  project_id BIGINT NOT NULL COMMENT '项目ID',
  tag_id BIGINT NOT NULL COMMENT '标签ID',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
  FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE,
  UNIQUE KEY uk_project_tag (project_id, tag_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='项目标签关联表';
```

#### 8. 文件资源表 (project_files)
```sql
CREATE TABLE project_files (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  project_id BIGINT NOT NULL COMMENT '项目ID',
  file_name VARCHAR(200) NOT NULL COMMENT '文件名',
  file_path VARCHAR(500) NOT NULL COMMENT '文件路径',
  file_size BIGINT COMMENT '文件大小（字节）',
  file_type VARCHAR(50) COMMENT '文件类型：需求文档/技术文档/设计稿/其他',
  mime_type VARCHAR(100) COMMENT 'MIME类型',
  description TEXT COMMENT '文件描述',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
  INDEX idx_project_id (project_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='项目文件表';
```

## 🚀 八、部署方案

### 8.1 开发环境
```bash
# 前端
cd blog
npm install
npm run dev

# 后端
cd backend-spring
mvn spring-boot:run
```

### 8.2 生产环境

**方式一：传统部署**
```bash
# 前端构建
npm run build
# 部署到Nginx

# 后端打包
mvn clean package
# 部署jar包
java -jar app.jar
```

**方式二：Docker部署**
```yaml
version: '3'
services:
  frontend:
    build: ./blog
    ports:
      - "80:80"
  
  backend:
    build: ./backend-spring
    ports:
      - "8080:8080"
    depends_on:
      - mysql
      - redis
  
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: your_password
      MYSQL_DATABASE: project_management
    volumes:
      - mysql_data:/var/lib/mysql
  
  redis:
    image: redis:7
    volumes:
      - redis_data:/data

volumes:
  mysql_data:
  redis_data:
```
