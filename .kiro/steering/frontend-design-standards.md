# 前端设计与开发统一规范


## 核心设计原则

### 可用性优先
输出**可直接落地的界面设计方案**，而非视觉炫技。遵循尼尔森十大可用性原则，确保：
- 降低认知成本，用户无需学习即可使用
- 通过设计预防错误，而非依赖提示
- 兼顾新手友好与高频用户效率
- 系统状态清晰可见，用户始终有控制感

### 视觉目标
- 根据页面目的选择合适的设计复杂度（工具型 vs 展示型）
- 避免千篇一律的设计，创造独特体验
- 清晰、克制、专业、可执行

## 二、尼尔森十大可用性原则（核心设计准则）

### 01. 系统状态可见性（Visibility of System Status）
**设计要求**：系统的当前状态必须在合理时间内清晰呈现，避免任何"不确定感"

**执行要点**：
- 明确展示 Loading / Processing / Success / Error 等状态
- 使用 Skeleton、进度条或即时反馈防止"假死感"
- 所有用户操作都应有即时响应

### 02. 系统与现实世界的匹配（Match Between System and the Real World）
**设计要求**：界面语言与逻辑应符合用户的现实经验，而非技术或内部视角

**执行要点**：
- 使用生活化、非技术化的文案
- 操作流程符合现实世界的因果顺序
- 图标、隐喻来源于日常经验

### 03. 用户控制与自由（User Control and Freedom）
**设计要求**：用户必须随时拥有"退出、撤销、回退"的控制权

**执行要点**：
- 清晰可见的 Back / Cancel / Close
- 支持撤销关键操作（如删除、提交）
- 避免强制不可逆流程

### 04. 一致性与标准（Consistency and Standards）
**设计要求**：相同概念在全产品中保持一致，遵循平台与行业规范

**执行要点**：
- 统一图标、按钮样式与交互反馈
- 相同功能使用相同文案和位置
- 遵循 iOS / Android / Web 设计规范

### 05. 防错设计（Error Prevention）
**设计要求**：通过设计提前避免错误，而不是事后报错

**执行要点**：
- 表单输入实时校验
- 明确规则与限制提示
- 对不可执行操作进行禁用或弱化

### 06. 识别优于回忆（Recognition Rather Than Recall）
**设计要求**：让用户"看到即可理解"，而非依赖记忆

**执行要点**：
- 关键选项可视化呈现
- 提供历史记录、最近使用
- 明确标签、辅助说明与提示

### 07. 灵活性与效率（Flexibility and Efficiency of Use）
**设计要求**：同一系统同时服务新手与专家用户

**执行要点**：
- 新手有清晰的默认路径
- 熟练用户可使用快捷方式
- 支持自定义与高级配置

### 08. 审美与简约设计（Aesthetic and Minimalist Design）
**设计要求**：只呈现当前任务所需的信息，避免视觉与信息噪音

**执行要点**：
- 合理留白，突出重点
- 清晰的信息层级
- 控制同屏信息密度

### 09. 帮助用户识别、诊断和恢复错误（Help Users Recognize, Diagnose, and Recover from Errors）
**设计要求**：错误提示必须"可理解 + 可行动"

**执行要点**：
- 使用自然语言描述错误（避免错误码）
- 明确指出原因
- 提供直接解决路径或按钮

### 10. 帮助与文档（Help and Documentation）
**设计要求**：帮助应在需要时出现，并快速解决问题

**执行要点**：
- 上下文式帮助与引导
- 示例、模板、操作演示
- 可随时关闭，不干扰主流程

## 三、页面设计与开发核心原则

### 1. 数据显示原则
- **不要自作主张增加数据**：在重构或修改页面时，严格按照原页面显示的数据进行设计，除非用户明确要求增加新数据
- 保持数据的一致性和准确性

### 2. 色彩搭配原则
- **契合项目整体风格**：这是最重要的原则
- 不要使用过于鲜艳的颜色
- 保持色彩的协调性和专业性
- 除非用户明确要求，否则必须考虑色彩搭配
- **⚠️ 必须确保文字和图标清晰可读**：这是设计的底线要求

**色彩系统设计**：
- **主色调**：建立品牌识别，占据主导地位
- **强调色**：用于关键操作和重点内容，形成视觉冲击
- **中性色**：用于背景和辅助信息
- **文字色彩**：确保与背景有足够对比度，优先考虑可读性

### 3. 布局设计原则
- **占满设计区域**：宽高都要充分利用
- 留下合适的间距，但不要超过设计高度
- **一屏显示所有内容**：尽量让用户无需上下左右滑动就能看到所有内容
- 多卡片页面时，相互之间上下左右一定要对齐

**创意布局技巧**（适用于展示页面）：
- **不对称布局**：打破平衡，创造动态感
- **元素重叠**：增加层次和深度
- **负空间运用**：宽松留白或受控密度
- **视觉对齐**：保持视觉的整齐和统一

### 4. 用户体验原则
- **站在用户角度思考**：如何操作最方便、最不容易犯错
- 操作流程要简洁高效
- 显示内容要清晰明了
- 减少用户的认知负担

### 5. 动画与交互
- 适当增加动画过渡效果，让页面看起来更加丝滑流畅
- 不要过度使用动画，保持专业感
- **优先使用纯 CSS 动画**：性能更好，实现更简洁
- **Vue 项目**：使用 Vue 的 transition 组件

**高冲击力动画时刻**：
1. **页面加载动画**：精心编排的入场效果
2. **错开揭示**：使用 animation-delay 创造层次感
3. **滚动触发**：基于滚动位置的动画效果
4. **悬停状态**：微妙但令人惊喜的交互反馈
