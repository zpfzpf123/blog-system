# AI服务配置说明

本项目集成了多种免费的AI服务，用于智能生成博客文章摘要。以下是详细的配置说明和使用指南。

## 🚀 支持的AI服务

### 1. 本地规则生成 (默认)
- **费用**: 完全免费
- **特点**: 无需网络连接，使用内置规则生成摘要
- **适用场景**: 离线使用、快速测试、无API密钥时
- **配置**: 无需配置

### 2. Hugging Face
- **费用**: 每月免费30,000次请求
- **特点**: 开源模型，支持多种语言，免费额度大
- **适用场景**: 个人博客、学习研究、开源项目
- **配置要求**: 需要注册账号并获取API密钥

#### 配置步骤：
1. 访问 [Hugging Face](https://huggingface.co/settings/tokens)
2. 注册账号并登录
3. 进入 Settings → Access Tokens
4. 创建新的Token
5. 复制Token到配置中

#### 推荐模型：
- `microsoft/DialoGPT-medium` - 对话生成，中文友好
- `THUDM/chatglm-6b` - 中文对话模型
- `fnlp/moss-moon-003-base` - 中文基础模型

### 3. Ollama (本地部署)
- **费用**: 完全免费
- **特点**: 本地运行，无需网络，支持多种开源模型
- **适用场景**: 隐私要求高、网络不稳定、长期使用
- **配置要求**: 需要下载并安装Ollama

#### 安装步骤：
1. 访问 [Ollama官网](https://ollama.ai/)
2. 下载对应系统的安装包
3. 安装并启动服务
4. 下载模型：`ollama pull llama2`

#### 常用模型：
- `llama2` - Meta开源模型，英文为主
- `qwen` - 阿里通义千问，中文友好
- `chatglm` - 清华ChatGLM，中文对话
- `codellama` - 代码生成专用

### 4. OpenAI
- **费用**: 新用户有免费额度，之后按使用量收费
- **特点**: 模型质量高，API稳定，支持多种语言
- **适用场景**: 商业项目、高质量要求、有预算
- **配置要求**: 需要信用卡验证

#### 配置步骤：
1. 访问 [OpenAI Platform](https://platform.openai.com/api-keys)
2. 注册账号并验证信用卡
3. 创建API密钥
4. 复制密钥到配置中

#### 推荐模型：
- `gpt-3.5-turbo` - 性价比高，响应快
- `gpt-4` - 质量最高，适合复杂任务
- `gpt-4-turbo` - 平衡质量和速度

### 5. Claude
- **费用**: 有免费使用额度
- **特点**: 安全性好，内容审核严格，支持中文
- **适用场景**: 企业使用、内容安全要求高
- **配置要求**: 需要申请访问权限

#### 配置步骤：
1. 访问 [Anthropic Console](https://console.anthropic.com/)
2. 申请访问权限
3. 创建API密钥
4. 复制密钥到配置中

## ⚙️ 配置方法

### 在博客创建页面配置：
1. 点击摘要字段旁边的"配置"按钮
2. 选择AI服务提供商
3. 填写相应的配置信息
4. 点击"测试连接"验证配置
5. 保存配置

### 配置项说明：
- **API密钥**: 用于身份验证的密钥
- **服务地址**: API服务的地址（Ollama需要）
- **模型名称**: 使用的AI模型名称

## 🔧 故障排除

### 常见问题：

#### 1. Hugging Face API调用失败
- 检查API密钥是否正确
- 确认模型名称是否存在
- 检查网络连接是否正常

#### 2. Ollama连接失败
- 确认Ollama服务是否启动
- 检查端口11434是否被占用
- 验证模型是否已下载

#### 3. OpenAI/Claude调用失败
- 检查API密钥是否有效
- 确认账户余额是否充足
- 检查网络代理设置

### 调试方法：
1. 查看浏览器控制台错误信息
2. 使用"测试连接"功能
3. 检查网络请求状态
4. 查看API服务状态页面

## 💡 使用建议

### 选择建议：
- **初学者**: 使用本地规则生成，无需配置
- **个人博客**: 推荐Hugging Face，免费额度大
- **隐私要求高**: 使用Ollama本地部署
- **商业项目**: 考虑OpenAI或Claude
- **中文内容**: 优先选择中文友好模型

### 成本控制：
- 合理设置摘要长度
- 避免频繁调用API
- 使用本地服务减少网络请求
- 选择合适的模型规格

### 性能优化：
- 缓存生成的摘要
- 批量处理多篇文章
- 使用异步调用避免阻塞
- 设置合理的超时时间

## 🔒 安全注意事项

1. **API密钥安全**:
   - 不要在代码中硬编码密钥
   - 定期更换密钥
   - 设置适当的权限范围

2. **数据隐私**:
   - 了解AI服务的数据使用政策
   - 避免上传敏感信息
   - 考虑使用本地服务保护隐私

3. **使用限制**:
   - 遵守API使用条款
   - 控制调用频率
   - 监控使用量避免超额
